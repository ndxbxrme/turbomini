Write apps, not dependency autopsies

Turbomini is for developers who want modern SPA ergonomics without the framework tax. Zero runtime dependencies, web components first, and a surface area you can actually hold in your head.

  • <3kb min+gzip core
  • No build step required
  • Optional router + store 
// hello-turbomini.html
<!doctype html>
<html lang="en">
  <body>
    <page></page>
    <script type="module">
      import { TurboMini } from './turbomini';
      const app = TurboMini('/');
      app.template('home', '<h1>Hello {{name}}</h1>');
      app.controller('home', () => ({ name: 'TurboMini' }));
      app.start();
    </script>
  </body>
</html>

Why Turbomini?

Designed for devs who are done with brittle dependency graphs and ever-expanding framework tax.

Tiny Core

Under 3kb of runtime. Enough to build real SPAs without the overhead.

Zero Dependencies

No external runtime packages to go stale, break, or get yanked.

Web Components First

Ship standards-based components that work anywhere.

Optional Routing

Add routing only if you need it — no mandatory framework structure.

Simple State

A tiny reactive store that doesn’t turn your app into a science project.

Fast by Default

Minimal abstraction means fewer surprises and quicker apps.

No Build Step

Use as ES modules today. Add tooling later if you want.

Friendly API

Fewer concepts, less ceremony, more shipping.
